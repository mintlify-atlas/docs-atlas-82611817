---
title: "/lint: health-check your wiki"
description: "Health-check the wiki: find orphan pages, contradictions, stale claims, missing pages, broken links, and data gaps. Fixes what's safe and reports the rest."
---

`/lint` runs the full lint workflow defined in `CLAUDE.md`. As your wiki grows across dozens of ingest sessions, links break, pages go unlinked, newer sources quietly supersede older claims, and topics accumulate without enough source coverage. Running `/lint` periodically surfaces all of these issues in one pass. Claude fixes what it safely can automatically and flags everything else for your review, then appends a summary to the log.

## Six health checks

<AccordionGroup>
  <Accordion title="Orphan pages">
    **What it finds:** Wiki pages that no other wiki page links to via `[[wikilinks]]`.

    Claude globs `wiki/*.md`, reads each page, and checks whether any other page links back to it. Orphan pages are either stubs that haven't been woven into the wiki yet, or pages that have been superseded and should be deleted.

    **Fix applied:** Claude adds links from relevant topic pages where appropriate. Pages with no clear home are flagged for you to decide.
  </Accordion>
  <Accordion title="Unresolved contradictions">
    **What it finds:** Pages that contain `[!WARNING] Contradiction` callouts that haven't been resolved since they were flagged.

    Claude greps `wiki/` for these callouts and reports each one: which pages are involved, what the conflict is, and which sources are cited on each side.

    **Fix applied:** Claude flags each contradiction to you with a recommendation — a new source may resolve it, or you may need to update one side of the claim.
  </Accordion>
  <Accordion title="Stale claims">
    **What it finds:** Claims on older pages that newer source-summary pages have superseded.

    Claude cross-references `updated` dates in page frontmatter against the dates of the source-summary pages in their `sources` list. A claim is considered potentially stale when a newer source covers the same topic and may contradict or update it.

    **Fix applied:** Claude updates the claim with the newer information, updates the `sources` frontmatter attribution, and updates the `updated` date.
  </Accordion>
  <Accordion title="Missing pages">
    **What it finds:** Concepts or entities referenced frequently via `[[wikilinks]]` that do not have their own wiki page.

    Claude extracts all `[[...]]` links from `wiki/*.md`, checks each against the file system, and surfaces links that resolve to nothing and appear in multiple pages.

    **Fix applied:** Claude creates stub pages for high-frequency missing links and queues them for future ingestion.
  </Accordion>
  <Accordion title="Broken links">
    **What it finds:** `[[wikilinks]]` that point to files that do not exist — including typos and pages that were renamed or deleted.

    Claude checks every wikilink in the wiki against the actual file system.

    **Fix applied:** Claude either creates a stub for the missing page, or corrects the link if the target file exists under a slightly different name.
  </Accordion>
  <Accordion title="Data gaps">
    **What it finds:** Important topics in the wiki that have many cross-references but thin source coverage — topic or entity pages with an empty or sparse `sources` frontmatter list.

    **Fix applied:** Claude reports each gap and suggests specific sources, articles, or web searches to fill it. No automatic edits are made for data gaps — filling them requires new ingests.
  </Accordion>
</AccordionGroup>

## Lint report

After running all six checks, Claude produces a report with three sections:

1. **Issues found** — count and description per category.
2. **Fixes applied** — every change made automatically during the lint run.
3. **Next sources to investigate** — recommended reads to address data gaps or resolve contradictions.

Claude then appends a summary to `wiki/log.md` (never overwrites it):

```
## [YYYY-MM-DD] lint | Lint run

N issues found, N fixed. [Brief summary of notable findings.]
```

## Running a focused check

You can pass an optional focus area to limit the lint run. Examples:

```
/lint broken links only
/lint check contradictions
/lint orphan pages
```

Leave the argument blank for a full check across all six categories.

<Note>
  `/lint` is a **vault-local** skill installed at `.claude/skills/lint/SKILL.md` inside your vault. It only runs in Claude Code sessions started from the vault directory.
</Note>
