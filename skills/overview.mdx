Skills are Claude Code slash commands that ship with every claude-second-brain vault. Instead of manually prompting Claude each time, you invoke a named command — Claude loads the skill's instructions and runs a structured, repeatable workflow. No copy-pasting, no forgotten steps.

## Daily workflow

<CardGroup cols={2}>
  <Card title="/brain-ingest" icon="file-import" href="/skills/brain-ingest">
    Drop a file into `raw-sources/` and run `/brain-ingest`. Claude reads the source, writes a summary page, updates related topic pages, and flags contradictions.
  </Card>
  <Card title="/brain-search" icon="magnifying-glass" href="/skills/brain-search">
    Ask anything. Claude runs hybrid search, reads the most relevant pages, and returns a cited answer with inline `[[wiki/page]]` links.
  </Card>
  <Card title="/lint" icon="circle-check" href="/skills/lint">
    Health-check the wiki. Surfaces orphan pages, broken links, unresolved contradictions, stale claims, and data gaps.
  </Card>
</CardGroup>

## Maintenance

<CardGroup cols={2}>
  <Card title="/brain-refresh" icon="arrows-rotate" href="/skills/brain-refresh">

## Skill scope

Three skills install globally so they are available in every Claude Code session. The rest live inside the vault and are specific to it.

| Skill | Scope | Install path | Description |
|---|---|---|---|
| `/brain-ingest` | Global | `~/.claude/skills/brain-ingest/` | Ingest a source into the wiki |
| `/brain-search` | Global | `~/.claude/skills/brain-search/` | Query the wiki with cited answers |
| `/brain-refresh` | Global | `~/.claude/skills/brain-refresh/` | Re-index changed files and regenerate embeddings |
| `/brain-rebuild` | Vault-local | `.claude/skills/brain-rebuild/` | Redesign the qmd schema from scratch |
| `/lint` | Vault-local | `.claude/skills/lint/` | Health-check the wiki for six categories of issues |
| `/setup` | Vault-local | `.claude/skills/setup/` | Initialize qmd collections for a new vault |
| `/qmd-cli` | Vault-local | `.claude/skills/qmd-cli/` | Direct access to qmd CLI commands from within Claude Code |

## Install or update skills

Skills are scaffolded into your vault at creation time. To pull in improvements from the latest template, use `npx skills` pointing to the template directory:

```bash
# Install or update all 7 wiki skills from the latest template
npx skills add https://github.com/jessepinkman9900/claude-second-brain/tree/main/template/.claude/skills -a claude-code -y

# Or update a specific skill
npx skills add https://github.com/jessepinkman9900/claude-second-brain/tree/main/template/.claude/skills --skill brain-ingest -a claude-code -y
```

Once installed via `npx skills`, future updates are a single command:

```bash
npx skills update -a claude-code
```

<Note>
  Global skills (`brain-ingest`, `brain-search`, `brain-refresh`) are installed to `~/.claude/skills/` during vault setup and work in any Claude Code session — not just inside your vault directory.
</Note>
