claude-second-brain turns years of reading into a living, cross-linked wiki. One `npx` command scaffolds the full stack: Claude Code ingests your sources and maintains the wiki, qmd powers local hybrid search, the vault opens directly in Obsidian, and GitHub keeps everything versioned and accessible from anywhere.

## What makes this different

This is not a RAG system or a chatbot over your notes. It's an **actively maintained, cross-linked wiki** — Claude reads every source, extracts what matters, files it into structured pages, and cross-links it to everything you already know.

## The stack

| Tool | Role |
|---|---|
| **Claude Code** | Reads sources, writes wiki pages, cross-links, flags contradictions |
| **qmd** | Local hybrid search (vector + BM25) across your entire wiki |
| **Obsidian** | Graph view, backlinks, mobile reading — offline, no extra sync |
| **GitHub** | Source of truth — version history, Claude Code anywhere, Obsidian sync |

## Get started in one command

```bash
npx claude-second-brain
```

<Steps>
  <Step title="Scaffold">
    Run `npx claude-second-brain`. The CLI asks for a folder name, qmd index path, and optionally creates a private GitHub repo and pushes automatically.
  </Step>
  <Step title="Initialize">
    `cd my-brain && claude`, then run `/setup`. Registers qmd collections and generates local vector embeddings. Downloads ~2 GB of GGUF models once.
  </Step>
  <Step title="Open in Obsidian">
    Open `my-brain/` as a vault in Obsidian. The Git plugin is pre-configured — enable it for automatic sync on every commit.
  </Step>
</Steps>
